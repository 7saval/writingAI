# Example: Project Showcase Section for Developer Portfolio

다음은 `frontend-design` 스킬을 사용할 때 참고할 수 있는 실제 예시 요청이다.

---

## User Request Example

너는 프로덕션 수준의 프론트엔드 디자이너이자 개발자다.

다음 요구사항을 만족하는 React 컴포넌트를 만들어라.

### Basic Requirements

- 컴포넌트명: ProjectShowcaseSection
- 목적: 개발자 포트폴리오에서 대표 프로젝트와 핵심 성과를 강하게 보여주는 섹션
- 주요 사용자: 채용 담당자, 면접관, 실무 엔지니어
- 사용 맥락: 포트폴리오 메인 페이지 또는 프로젝트 소개 페이지에서 사용되며, 사용자가 짧은 시간 안에 대표 프로젝트의 수준과 강점을 파악해야 하는 구간
- 반드시 포함할 콘텐츠:
  - 섹션 타이틀
  - 대표 프로젝트 1개
  - 서브 프로젝트 2~3개
  - 프로젝트 설명
  - 사용 기술 스택
  - 핵심 성과 또는 임팩트 수치
  - 상세보기 또는 링크 버튼
- 필수 기능:
  - 대표 프로젝트가 시각적으로 가장 먼저 보일 것
  - hover 인터랙션
  - 반응형 대응
  - 키보드 접근 가능
  - 모바일에서도 정보 우선순위가 무너지지 않을 것
- 사용 기술 스택: React + TypeScript + Tailwind
- 반응형 필요 여부: 필요
- 접근성 요구사항:
  - 시맨틱 구조 사용
  - 링크와 버튼의 포커스 스타일 명확히 제공
  - 텍스트 대비 확보
  - hover에만 의존하지 않는 정보 전달
- 성능 고려사항:
  - 불필요한 리렌더링이 없는 구조
  - 무거운 애니메이션 지양
  - CSS 기반 인터랙션 우선
  - 이미지가 있다면 과도한 레이아웃 시프트를 유발하지 않을 것

### Design Direction

- 전체 톤앤매너: sharp editorial tech / cold premium / cinematic
- 참고할 감성 키워드:
  - technical
  - precise
  - premium
  - structured
  - tense
  - intelligent
- 피해야 할 느낌:
  - 흔한 카드 나열형 포트폴리오
  - 스타트업 템플릿 같은 랜딩 섹션
  - 과도하게 둥글고 귀여운 UI
  - generic SaaS dashboard aesthetics
  - 보라색 그라디언트 중심 AI 스타일
- 가장 기억에 남아야 할 포인트:
  - 대표 프로젝트 1개가 압도적으로 돋보이는 계층 구조
  - 제목, 성과 수치, 기술 정보가 강한 리듬감으로 배치되는 editorial composition
  - hover 시 정보 레이어가 정교하게 반응하는 카드 경험
- 컬러 무드:
  - black
  - graphite
  - silver gray
  - off-white
  - muted blue or acid lime accent (절제해서 사용)
- 타이포그래피 방향:
  - 강한 제목 대비
  - 작은 메타 정보 타이포가 정교하게 정렬된 구조
  - 밀도감 있는 정보 배치
  - 기술적인 인상을 주되 기계적이지만은 않게
- 레이아웃 성격:
  - 잡지 편집 같은 비대칭 구성
  - 대표 카드와 보조 카드의 크기 차이를 분명히 둘 것
  - 정보 블록 간 긴장감 있는 간격 사용
  - 그리드에 기대되, 완전히 평범하지 않은 배치
- 애니메이션 방향:
  - 초기 진입 시 순차적 reveal
  - hover 시 텍스트 또는 정보 레이어가 미세하게 이동
  - 빠르고 가벼운 전환
  - 지나치게 화려하지 않지만 세밀한 반응
- 배경 또는 장식 디테일:
  - 지나치게 화려한 배경 금지
  - 어두운 배경 위 미세한 texture 허용
  - 얇은 divider line, subtle glow, information frame 활용 가능
  - 기술 문서 또는 매거진 편집 같은 긴장감을 주는 요소 사용

### Output Requirements

- 바로 실행 가능한 코드로 작성
- 더미 데이터 포함
- 한 파일로 작성
- Tailwind만 사용
- 코드에 핵심 의도를 주석으로 설명
- 실제 포트폴리오에 붙여넣을 수 있을 정도로 완성도 있게 구현
- 디자인이 “템플릿 같다”는 느낌이 들지 않게 작성
- 정보량이 많아도 시선 흐름이 명확해야 함

### My Request

대표 프로젝트는 아래 내용을 사용해줘.

- 프로젝트명: 모아클래스
- 한 줄 설명: 지역 기반 원데이 클래스 탐색 및 예약 플랫폼
- 상세 설명: 사용자가 지역, 카테고리, 일정 기준으로 클래스를 탐색하고 예약할 수 있도록 설계한 플랫폼. 검색 유입과 콘텐츠 탐색 경험을 동시에 고려해 상세 페이지 중심 구조와 반응형 UI를 설계함.
- 기술 스택: Next.js, React, TypeScript, Tailwind CSS, React Query
- 핵심 성과:
  - Lighthouse 성능 지표 95점 달성
  - 상세 페이지 SEO 구조 개선
  - 반응형 UI 설계 및 사용자 흐름 개선

## Why This Example Works

이 예시는 다음 요소를 모두 포함한다.

- 대표 프로젝트의 우선순위가 분명하다
- 포트폴리오 사용 맥락이 구체적이다
- 정보 구조와 디자인 방향이 동시에 제시된다
- 단순 카드 나열을 피하도록 명확히 제한한다
- 기술 정보와 성과 정보가 함께 드러난다
- 구현 시 어떤 긴장감과 리듬을 만들어야 하는지까지 전달한다

즉, 이 예시는  
단순히 “프로젝트 카드 만들어줘”가 아니라  
**포트폴리오 맥락에서 무엇을 강조해야 하는지까지 포함한 요청**이다.

## Reusable Pattern

이 예시는 아래와 같은 포트폴리오 UI에도 응용할 수 있다.

- 대표 프로젝트 상세 섹션
- 경력 요약 섹션
- 기술 스택 소개 섹션
- 성과 중심 케이스 스터디 레이아웃
- 협업 경험 아카이브 섹션

핵심은 아래 5가지를 명확히 적는 것이다.

1. 어떤 프로젝트를 가장 먼저 강조할지
2. 누가 이 화면을 보는지
3. 어떤 정보가 반드시 보여야 하는지
4. 어떤 시각적 리듬과 분위기를 원하지
5. 무엇을 절대 피해야 하는지
