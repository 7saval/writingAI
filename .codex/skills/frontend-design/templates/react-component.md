## Writing Guidance

좋은 결과를 원한다면 아래 항목을 반드시 구체적으로 작성하라.

### A. Purpose

단순히 “카드 만들어줘”라고 하지 말고,
이 컴포넌트가 어떤 화면에서 어떤 역할을 하는지 설명하라.

예시:

- 개발자 포트폴리오에서 대표 프로젝트를 강조하는 섹션
- 원데이 클래스 플랫폼의 메인 랜딩 히어로 영역
- 관리자 대시보드의 실시간 상태 요약 패널

### B. Audience

누가 보는지 명확히 써라.

예시:

- 채용 담당자
- 20~30대 취미 탐색 사용자
- 내부 운영팀 담당자

### C. Tone

디자인 방향은 반드시 분명해야 한다.

예시:

- editorial / magazine
- luxury / refined
- retro-futuristic
- brutalist / raw
- soft / organic
- industrial / utilitarian

### D. Avoidances

결과물이 흔해지지 않으려면 “피해야 할 느낌”을 꼭 써라.

예시:

- 평범한 SaaS 대시보드 느낌 금지
- 흔한 카드 나열형 레이아웃 금지
- 과한 보라색 그라디언트 금지
- generic startup landing page 느낌 금지

### E. Signature Moment

“이 디자인에서 가장 기억에 남아야 할 한 가지”를 명시하라.

예시:

- 타이포그래피가 첫인상을 압도해야 함
- 카드 hover 시 정보 레이어가 밀려 올라와야 함
- 비대칭 레이아웃이 핵심 인상이어야 함
- 입장 애니메이션이 강하게 기억에 남아야 함

## Minimal Prompt Version

시간이 없을 때는 아래 형식만 써도 된다.

React + Tailwind로 [무엇] 만들어줘.  
용도는 [목적]이고, 사용자는 [누구]야.  
분위기는 [톤]으로 가고, [피해야 할 느낌]은 절대 쓰지 마.  
가장 중요한 포인트는 [기억 포인트]야.  
반응형 포함해서, 바로 실행 가능한 코드로 작성해줘.

## Example Prompt 1

React + Tailwind로 포트폴리오용 프로젝트 소개 섹션 만들어줘.  
용도는 채용 담당자가 대표 프로젝트를 빠르게 이해하게 하는 것이고, 사용자는 면접관과 채용 담당자야.  
분위기는 차갑고 세련된 editorial tech 무드로 가고, 흔한 카드 나열형 포트폴리오 느낌은 절대 쓰지 마.  
가장 중요한 포인트는 대표 프로젝트 1개가 압도적으로 돋보이는 계층형 레이아웃이야.  
반응형 포함해서, 바로 실행 가능한 코드로 작성해줘.

## Example Prompt 2

React + Tailwind로 랜딩페이지 히어로 섹션 만들어줘.  
용도는 지역 기반 원데이 클래스 플랫폼 메인 첫 화면이고, 사용자는 취미와 자기계발에 관심 있는 20~30대야.  
분위기는 warm editorial lifestyle 무드로 가고, 흔한 스타트업 SaaS 랜딩 느낌은 절대 쓰지 마.  
가장 중요한 포인트는 비대칭 레이아웃과 강한 타이포그래피야.  
반응형 포함해서, 바로 실행 가능한 코드로 작성해줘.

## Final Reminder

항상 다음을 기억하라.

- 목적이 명확해야 한다
- 사용자 맥락이 보여야 한다
- 디자인 방향이 강해야 한다
- 피해야 할 느낌을 분명히 써야 한다
- 코드가 아니라 “디자인된 결과물”을 요청해야 한다
