# Example: Editorial Hero Section for a Class Platform

다음은 `frontend-design` 스킬을 사용할 때 참고할 수 있는 실제 예시 요청이다.

---

## User Request Example

너는 프로덕션 수준의 프론트엔드 디자이너이자 개발자다.

다음 요구사항을 만족하는 React 컴포넌트를 만들어라.

### Basic Requirements

- 컴포넌트명: ClassHeroSection
- 목적: 원데이 클래스 플랫폼의 메인 랜딩페이지 첫 화면
- 주요 사용자: 20~30대 취미, 자기계발, 오프라인 소셜 경험에 관심 있는 사용자
- 사용 맥락: 사용자가 서비스에 처음 진입했을 때 클래스 탐색 욕구를 자극하는 가장 중요한 첫 화면
- 반드시 포함할 콘텐츠:
  - 메인 카피
  - 서브 카피
  - CTA 버튼 2개
  - 대표 비주얼 영역
  - 짧은 통계 또는 신뢰 요소
- 필수 기능:
  - CTA hover 인터랙션
  - 반응형 대응
  - 진입 애니메이션
  - 모바일에서도 타이포 인상이 유지될 것
- 사용 기술 스택: React + Tailwind
- 반응형 필요 여부: 필요
- 접근성 요구사항:
  - 시맨틱 태그 사용
  - 버튼 대비 확보
  - 키보드 포커스 스타일 포함
- 성능 고려사항:
  - 과도한 JS 애니메이션 지양
  - CSS 중심 모션 사용
  - 첫 화면 렌더링에 부담이 큰 효과는 피할 것

### Design Direction

- 전체 톤앤매너: editorial / magazine + warm premium lifestyle
- 참고할 감성 키워드:
  - curated
  - cultural
  - tactile
  - warm
  - premium
  - lifestyle
- 피해야 할 느낌:
  - 흔한 스타트업 SaaS 랜딩페이지
  - 보라색 그라디언트 중심 AI 느낌
  - 뻔한 중앙 정렬 hero section
  - generic app marketing page
- 가장 기억에 남아야 할 포인트:
  - 비대칭 레이아웃
  - 강한 타이포그래피
  - 이미지와 텍스트가 리드미컬하게 겹치는 구성
- 컬러 무드:
  - cream
  - charcoal
  - muted gold
  - deep orange accent
- 타이포그래피 방향:
  - 존재감 있는 디스플레이 타이포
  - 차분하고 고급스러운 본문 타이포
  - 제목과 본문 대비가 분명할 것
- 레이아웃 성격:
  - 넓은 여백
  - 비대칭 구성
  - 잡지 편집 같은 시선 흐름
  - 일부 요소의 겹침 허용
- 애니메이션 방향:
  - 페이지 진입 시 staggered reveal
  - hover 시 조용하지만 정교한 반응
  - 시선을 방해하지 않는 수준의 움직임
- 배경 또는 장식 디테일:
  - 단색 배경 금지
  - 아주 미세한 grain 또는 texture
  - 얕은 레이어드 배경
  - 얇은 선 장식 또는 editorial divider 활용

### Output Requirements

- 바로 실행 가능한 코드로 작성
- 더미 데이터 포함
- 한 파일로 작성
- Tailwind만 사용
- 코드에 핵심 의도를 주석으로 설명
- 실제 서비스 랜딩페이지에 넣을 수 있을 정도의 디테일로 구현
- 흔한 템플릿 느낌이 나지 않게 작성
- 폰트는 Inter, Roboto, Arial 같은 뻔한 선택을 기본 전제로 삼지 말 것

### My Request

메인 카피는  
“가까운 동네에서, 취향이 맞는 클래스를 만나보세요”  
로 작성해줘.

서브 카피는  
“배움은 더 이상 멀리 있지 않습니다. 공방, 쿠킹, 드로잉, 운동까지. 오늘의 기분에 맞는 클래스를 지금 탐색해보세요.”  
로 작성해줘.

CTA 버튼은

- “클래스 둘러보기”
- “호스트 시작하기”
  두 개를 넣어줘.

신뢰 요소는 아래 중 3개 정도를 자연스럽게 보여줘.

- 누적 클래스 1,200+
- 재방문율 68%
- 평균 만족도 4.8/5
- 전국 42개 지역 진행

## Why This Example Works

이 예시는 다음 요소를 모두 포함한다.

- 무엇을 만드는지 명확하다
- 어떤 사용자를 위한 화면인지 분명하다
- 디자인 방향이 강하다
- 피해야 할 느낌이 분명하다
- 기능과 기술 제약이 함께 제시된다
- “기억에 남아야 할 포인트”가 명확하다

즉, 단순한 “예쁜 UI” 요청이 아니라  
**맥락 + 미학 + 구현 조건**이 함께 들어간 좋은 요청 예시다.

---

## Reusable Pattern

이 예시를 다른 컴포넌트에도 그대로 응용할 수 있다.

예:

- 포트폴리오 프로젝트 섹션
- 제품 소개 카드 영역
- 로그인 화면
- 대시보드 헤더
- 프로모션 배너
- 가격표 섹션

핵심은 아래 5가지를 빠짐없이 적는 것이다.

1. 목적
2. 사용자
3. 필수 기능
4. 디자인 방향
5. 피해야 할 느낌
