# Dev-Logger Skill

사용자가 "개발 로그 써줘", "오늘 요약해줘", "데브로그 업데이트 해줘" 등의 요청을 할 경우 다음 절차를 엄격히 따르세요:

1. **가이드 숙지**: `_docs/dev-log/README.md` 파일을 읽고 마크다운 양식, 이모지, 체크박스 규칙을 파악하세요.
2. **오늘(Date)의 기준 명확화**: 가이드에서 말하는 '오늘'은 기획 문서 상의 주차(Phase)나 열려있는 문서가 아닌, **AI가 현재 접속해 있는 시스템 상의 실제 달력 날짜(Calendar Date)**를 의미합니다.
3. **신규 생성 및 업데이트 판단**: `_docs/dev-log/` 폴더를 확인하여, 만약 **수행하는 당일(실제 날짜)에 해당하는 차수의 파일(예: 최고차수 `dayN.md`가 당일 생성된 경우)이 존재할 경우 새 파일을 만들지 말고**, 오늘 새로 진행한 내용(완성된 기능, 해결한 문제 등)을 **기존 문서에 보완/추가**하세요. 만약 오늘의 실제 날짜에 생성된 파일이 없다면 새 파일 `day{N+1}.md`를 생성하세요.
4. **컨텍스트 요약**: 새로 대화한 목표 설정, 배운 점, 해결한 에러 등을 추출합니다. 기존 내용의 누락 없이 내용을 적절히 융합하여 적으세요.
5. **파일 작성/업데이트**: `_docs/dev-log/dayN.md` 파일에 README.md의 템플릿 구조를 100% 준수하여 한국어로 작성 및 업데이트합니다.
