# Example: Performance Optimization Case

## Input Summary

- 문제: 리뷰 입력 시 전체 모달 리렌더링 발생
- 해결: useWatch + 컴포넌트 분리
- 결과: 입력 반응성 개선

---

## Output

### 한 줄 요약

폼 상태 구독 범위를 하위 컴포넌트로 분리해 입력 시 불필요한 리렌더링을 줄였습니다.

### 상세 설명

리뷰 작성 모달에서 입력 이벤트 발생 시 전체 컴포넌트가 리렌더링되는 문제가 있었습니다. 이를 해결하기 위해 실시간 변경이 필요한 UI만 분리하고, 해당 영역에서만 상태를 구독하도록 설계하여 렌더링 범위를 축소했습니다.

### 기술 포인트

- useWatch 활용
- 컴포넌트 분리
- 상태 구독 범위 제한

### 성과 강조

입력 시 불필요한 렌더링을 제거해 사용자 입력 반응성을 개선했습니다.
